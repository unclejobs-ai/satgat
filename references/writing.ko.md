# 한국어 카피 가이드

> AI가 한지 위에 옮겨 적을 때 따를 톤·문체·어휘 표준.

---

## 1. 기본 톤

**단정·격조·정직.**

- 단정문 우선. "~합니다" / "~입니다" / "~합니다."
- 영문 직역체 금지. "~하는 것입니다" → "~합니다"
- 한자(漢字) 보조 — 격을 갖춰야 할 자리에만 (履歷書 · 名銜 · 提案書)
- 과장 금지. "혁신적인 솔루션" → "이러한 방식"

---

| 영문 직역체 | 한국 단정문 |
|------------|------------|

## 3. 문체 구조

### 짧은 문장
```
좋은 콘텐츠는 좋은 종이를 만날 자격이 있습니다.
한지 위에 먹으로 옮겨 적습니다.
단청 한 점이면 충분합니다.
```

한 줄에 한 사실. 접속사("그리고"·"하지만"·"그러므로")는 절제.

### 한자 보조
```
이력서(履歷書)
명함(名銜)
규(規) — 규칙
인장(印章) — 도장
여백(餘白) — 빈 공간
```

처음 등장할 때 한자 괄호 표기. 같은 단락 내 반복 시 한자 생략.

---

## 4. 숫자·강조

### 숫자
- 단위 한국어: "10분" / "100억원" / "8가지" / "7년차"
- 천 단위 콤마: "12,400원" (한국 관례)
- 큰 숫자 한자: "삼백억" (격조 자리에만)
- 통화: "₩" 또는 "원" (혼용 금지, 한 문서 내 통일)

### 강조
- 단청 색으로 단어 하나만: `<em class="latin" style="color:var(--brand)">단청</em>`
- 굵게는 weight 700/800만 — 합성 볼드 금지
- 밑줄 절대 금지 (인쇄 시 잉크 번짐)

## 5. 인용·일러두기

### 단정문 인용
```
"한 사람의 자취를 한 장의 종이에 옮긴다.
 장식보다 명확함을, 화려함보다 절제를."
```

좌측 2pt 단청 바 + 숯(炭) 텍스트.

### 일러두기 (안내문)
짧고 단호하게:
```
※ 단청은 한 페이지의 5%를 넘지 않습니다.
※ 본문 행간 1.72 — 한글 가독성 기준.
```

---

## 6. 헤드라인 작성

### 패턴 A — 명사 두 개 + 동사
```
좋은 콘텐츠는 좋은 종이를 만납니다.
한 사람의 자취를 한 장의 종이에.
```

### 패턴 B — 한자 + 한국어
```
색(色) — 단청을 절제하다
자(字) — 명조가 위계다
규(規) — 여덟 가지 규칙
```

### 패턴 C — 짧은 명사구
```
여덟 종류의 종이
따뜻한 절제
결을 그리는 차트
```

---

## 7. 캡션·메타

### 캡션
한 줄. 단정문. 영문 직역체 금지.
```
✅ 시스템 구성요소와 연결. 단청은 중심 노드 하나에만.
❌ 시스템 구성 요소들의 연결관계를 보여줍니다.
```

### 메타 (label)
대문자 + letter-spacing 0.18em.
```
SECTION · 작 업
PROPOSAL · 2026
```

## 8. CTA (행동 유도)

### ❌
- "지금 시작하세요!"
- "무료로 사용해보세요!"
- "AI로 더 빠르게!"

### ✅
- "새 문서 생성"
- "전체 견본 보기"
- "한지 위에 옮겨 적기"

명령형보다 명사형. 동사 쓰더라도 단정형 ("적습니다" / "옮깁니다").

---

## 9. 에러 메시지

### ❌
- "오류가 발생했습니다. 다시 시도해주세요."
- "Something went wrong"

### ✅
- "한지 위에 옮기지 못했습니다. 다시 시도해 주세요."
- "이야기를 다시 풀어 적어 주세요. (현재 글자 수: 0)"

기술적 에러는 사용자가 이해 가능한 한국말로.

## 11. 절대 금지

1. ❌ 영문 직역체 ("~하는 것입니다")
2. ❌ 과장 형용사 ("혁신적인" / "획기적인" / "놀라운")
3. ❌ 명령형 CTA ("지금 시작!")
4. ❌ 외국 사례 (Tesla / Musk / Kaku)
5. ❌ 마침표 뒤 빈 줄 두 개 이상
6. ❌ 한 단락 5문장 초과
