# 이력서(履歷書) 작성 가이드

> 한국 채용 관습에 맞는 이력서를 한지 위에 옮겨 적기 위한 brief.

---

## 0. 한국 이력서 = 외국 résumé가 아니다

| | 외국 résumé | 한국 이력서 |
|---|------------|------------|
| 분량 | 1장 (엄격) | 1-2장 (가변) |
| 증명사진 | 금지 (차별 방지) | 우상단 (관례) |
| 한자 이름 | 없음 | 옵션 (격을 위해) |
| 순서 | 최신 경력 → 학력 | 인적사항 → 경력 → 학력 → 스킬 |
| 어조 | 동사 위주 ("Led", "Built") | 명사·체언 위주 ("리드", "구축") |
| 사이즈 | Letter (8.5×11") | A4 (210×297mm) |

---

## 1. 슬롯 정의

```typescript
interface ResumeSlots {
  // 필수
  name: string;              // 김상세
  hanja_name?: string;       // 金詳細 (옵션, 격조 자리)
  title: string;             // 프로덕트 디자이너 · 7년차
  email: string;
  phone?: string;

  // 사진 (옵션)
  photo_url?: string;        // 또는 placeholder
  
  // 본문
  summary?: string;          // 자기소개 (2-3 문장)
  experiences: Array<{
    period: string;          // "2024 — 現" / "2020 — 2024"
    company: string;         // "카카오"
    role: string;            // "시각디자인팀"
    description?: string;    // 한 줄 부연 (생략 가능)
  }>;
  educations: Array<{
    period: string;          // "2014 — 2018"
    school: string;          // "홍익대"
    degree: string;          // "시각디자인 학사"
  }>;
  skills: string[];          // 기술·도구 — comma-separated
  certifications?: Array<{
    name: string;
    date: string;
  }>;
}
```

## 2. 레이아웃 (A4 1-2장)

```
┌─────────────────────────────────────┐
│ ┌─────────┐  김상세 / 金詳細     │  ← 헤더
│ │ photo   │  프로덕트 디자이너 · 7년차│
│ │ 28×36   │  email · phone       │
│ └─────────┘                       │
│ ──────────────────── (먹 1.5pt) ──│
│                                   │
│ 자기소개(自己紹介)                │  ← 옵션
│ 한 단락, 2-3 문장.                │
│                                   │
│ 경력(經歷)                        │  ← 핵심
│ 2024 — 現    카카오 · 시각디자인팀│
│              [한 줄 부연]         │
│ 2020 — 2024  무신사 · UX 리드     │
│              ...                  │
│                                   │
│ 학력(學歷)                        │
│ 2014 — 2018  홍익대 · 시각디자인  │
│                                   │
│ 스킬(技能)                        │
│ Figma · React · TypeScript · ...  │
│                                   │
│ 자격(資格)                        │  ← 옵션
│ ...                               │
│                            [履]   │  ← 인장 우하단
└─────────────────────────────────────┘
```

## 3. 톤·문체

### 명사·체언 위주 (한국 관습)
```
✅ "카카오 · 시각디자인팀"
✅ "UX 리드 · 7년차"
❌ "I led the visual design team at Kakao"
❌ "카카오에서 시각디자인팀을 이끌었습니다"
```

### 부연 — 한 줄, 간결
```
✅ "한국형 PDP 디자인 시스템 구축 · CVR +18%"
❌ "다양한 한국형 PDP 디자인 시스템을 구축하여 CVR 향상에 기여한 경험이 있습니다"
```

### 기간 표기
```
✅ "2024 — 現"  (한자 現 사용 시 격조 ↑)
✅ "2024 — 현재"
✅ "2020 — 2024"
❌ "2020 - 2024"  (en-dash 권장)
❌ "Jan 2024 - Present"
```

## 4. 한자(漢字) 사용 — 격조 자리에만

| 자리 | 한자 보조 | 예시 |
|------|----------|------|
| 이름 | 옵션 | 김상세 / 金詳細 |
| 섹션 제목 | 권장 | 경력(經歷) · 학력(學歷) · 스킬(技能) |
| 본문 | 금지 | (오버 디자인) |
| 회사·학교명 | 절대 금지 | (실명 그대로) |
| 인장 | 필수 | 履 (이력서) |

---

## 5. 증명사진 자리

```
크기: 28×36mm (실제 한국 표준 사진 크기 축소)
위치: 우상단
배경: 황지(#E8E1CC) placeholder 또는 실제 사진
테두리: 0.5pt 보더 옅음
```

사진이 없으면 비워두지 말고 `[증명사진 자리]` 흐릿한 placeholder.

## 6. 디자인 토큰

```css
/* 이력서 한정 미세 조정 */
--resume-photo-bg:    var(--warm-sand);   /* #E8E1CC */
--resume-rule:        var(--near-black);  /* 1.5pt 헤더 구분선 */
--resume-section-rule: var(--border-soft); /* 0.5pt 섹션 구분선 */
--resume-period:      var(--near-black);  /* 기간 강조 */
--resume-company:     var(--dark-warm);   /* 회사명 */
--resume-description: var(--olive);       /* 부연 */
```

## 7. 인장(印章) — 履

```
글자: 履 (이력의 履)
색: ink #1C1916 (먹색) — 격조
크기: 22-26pt
위치: 우하단 14px 안쪽
폰트: Nanum Myeongjo 800
사각 모서리: 3px
white inset glow: rgba(255,255,255,0.18) 1.2px
```

## 9. 자주 보는 실수

### ❌ 영문 직역 동사
```
"리드했음", "구축했음", "개선시켰음"
```

### ✅ 명사·체언
```
"리드 · 7년차", "구축", "개선 · CVR +18%"
```

---

### ❌ 너무 긴 부연
```
"카카오 시각디자인팀에서 한국형 PDP 디자인 시스템을 구축하고
CVR 향상에 기여하여 회사의 전환율 지표를 개선한 경험이 있습니다."
```

### ✅ 한 줄
```
"카카오 · 시각디자인팀 — 한국형 PDP 시스템 · CVR +18%"
```

---

### ❌ 두 페이지 가득 메우기
```
한 항목당 3-5줄 설명
```

### ✅ 여백 두기
```
한 항목당 1-2줄 — 면접에서 풀어 설명
```

## 10. 인쇄 검증

- A4 한 장에 모든 내용 들어가는지 확인
- 두 장이면 페이지 구분이 자연스러운지 (섹션 중간에 끊기지 않도록)
- 인장 우하단 위치 유지
- 한지 배경 끝까지 인쇄 (흰 테두리 없음)
